## Rust FFI 方案深度评估（与当前 Dart 实现对比）
- 详细评估了 MD5 / EXIF / thumbnail / batch 四个模块的替代方案
- **thumbnail (decoder.rs)**: 内存优化最大——libjpeg-turbo IDCT 缩放解码将单张峰值内存从 ~210MB 降到 ~120KB（降低 1750 倍），速度 10-20x
- **EXIF (exif.rs)**: 桌面端最大 CPU 收益——替换 587 行手写 Dart 二进制解析器为原生 kamadak-exif，10-30x 提速
- **MD5 (hasher.rs)**: 内存收益为主——消除 `readAsBytes()` 全量读入，改为 64KB 流式
- **rayon 并行**: 架构级改进——消除 Dart Isolate 消息复制开销
- **风险点**: kamadak-exif 字段覆盖度需验证、Rust panic 跨 FFI 边界、构建复杂度上升、移动端不受益
- 建议推进顺序：decoder → exif → hasher
