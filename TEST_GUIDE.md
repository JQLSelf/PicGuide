# 🧪 扫描算法测试指南

## 快速验证步骤

### 1️⃣ **启动应用并进入浏览器页面**
- 打开应用
- 点击"导入文件夹"
- 选择一个包含大量图片的文件夹（建议100+张）

### 2️⃣ **观察扫描配置弹窗**
你应该看到：
- ✅ 设备信息卡片（CPU核心数、内存等）
- ✅ 三种算法选项（🐢顺序、🚀并行、🤖自适应）
- ✅ 推荐标签（根据你的设备自动标注）

### 3️⃣ **测试三种算法**

#### **测试1：顺序扫描（最稳定）**
1. 选择 🐢 **顺序扫描**
2. 点击"开始扫描"
3. 观察控制台日志：
   ```
   🚀 开始扫描: /your/folder/path
   📊 算法配置: 顺序扫描 (Worker: 1, Batch: 1)
   ```
4. **预期行为**：
   - 进度条平稳增长
   - 文件逐个处理
   - 内存占用低

#### **测试2：并行扫描（最快）**
1. 再次导入同一文件夹
2. 选择 🚀 **并行扫描**
3. 点击"开始扫描"
4. 观察控制台：
   ```
   📊 算法配置: 并行扫描 (Worker: 8, Batch: 100)
   ```
5. **预期行为**：
   - 进度跳跃式增长（每批100个文件）
   - 扫描速度明显快于顺序模式
   - CPU使用率较高

#### **测试3：自适应扫描（智能）**
1. 再次导入同一文件夹
2. 选择 🤖 **自适应扫描**
3. 点击"开始扫描"
4. 观察控制台：
   ```
   📊 算法配置: 自适应扫描 (Worker: 4, Batch: 50)
   ⬆️ 性能良好，增加 Worker: 4 -> 5
   ⬇️ 性能下降，减少 Worker: 5 -> 4
   ```
5. **预期行为**：
   - Worker数量动态调整
   - 根据设备性能自动优化

### 4️⃣ **测试生命周期管理**

#### **暂停/恢复测试**
1. 在扫描过程中点击"暂停"按钮
2. 观察：
   - ✅ 进度停止增长
   - ✅ 按钮变为"恢复"
   - ✅ 控制台无新日志
3. 点击"恢复"
4. 观察：
   - ✅ 扫描继续
   - ✅ 进度正常更新

#### **停止测试**
1. 在扫描过程中点击"停止"
2. 如果出现确认对话框，点击"确认停止"
3. 观察：
   - ✅ 扫描立即中断
   - ✅ 进度覆盖层消失
   - ✅ 已处理的文件已保存

## 🐛 常见问题排查

### **问题1：并行模式没有加速**
**可能原因**：
- 设备CPU核心数太少（≤2核）
- 硬盘IO瓶颈（机械硬盘）
- 数据库锁竞争

**解决方案**：
- 使用SSD存储
- 减小batchSize（改为50或20）
- 切换到自适应模式

### **问题2：内存占用过高**
**可能原因**：
- 并行Worker数量太多
- 图片文件太大

**解决方案**：
- 降低workerCount到4
- 减小batchSize到50
- 使用顺序扫描

### **问题3：暂停后无法恢复**
**检查步骤**：
1. 查看控制台是否有错误
2. 确认ScanController状态
3. 检查是否有死锁

### **问题4：进度不准确**
**可能原因**：
- 并行模式下进度报告时机不对
- 异步操作未正确await

**当前实现已解决**：
- 每个批次完成后统一报告
- 避免重复计数

## 🎯 下一步优化方向

基于测试结果，可以考虑：

1. **Isolate隔离**：将CPU密集型任务放到独立Isolate
2. **优先级队列**：先处理小文件，快速显示进度
3. **增量扫描**：只处理新增/修改的文件
4. **断点续扫**：保存扫描位置，支持从断点继续
5. **预测性加载**：预读下一批文件的元数据
