# PicGuide 使用手册

> 一款**完全离线**的本地图片管理工具。
> 不依赖任何云服务、不会上传你的照片，所有数据都保存在你的电脑上。
>
> **v2.0 更新**：采用企业级 **类型_功能** 命名规范，代码可读性和维护性全面提升。
> **扫描引擎升级**：采用 Isolate 多核并行扫描（P0），非阻塞进度面板（P2），批量数据库预加载（P3），零 CPU 暂停机制（P4），缩略图内存安全控制（P5）。
> **v2.1 体验优化**：统一缩略图组件（双清晰度策略），详情/全屏页按钮布局重构（解决矮图重叠），缩略图内存管理。
> **v2.2 Rust FFI 原生加速（Windows）**：扫描、MD5、EXIF、缩略图解码等核心流程迁移到 Rust，Windows 桌面端利用多核多线程进一步提速，同时保持内存占用极低。移动端和其他平台仍保持 Dart 实现，无需额外配置。

## 1. 快速开始

### 1.1 主界面

应用启动后默认进入 **仪表盘**，左侧胶囊导航栏可切换：

- **浏览器** – 浏览、挑选、给照片打标签的主界面
- **仪表盘** – 查看总览、文件大小分布、拍摄城市分布、标签词云

顶部右侧 **`📖 使用手册`** 按钮随时打开本说明。

### 1.2 系统要求

| 项目 | 要求 |
| --- | --- |
| 操作系统 | Windows 10 / 11（64 位） |
| 磁盘空间 | 视照片库而定，仅数据库约 < 50 MB |
| 内存 | ≥ 4 GB |
| 网络 | **不需要**（首次使用建议连网，便于在线反查城市名） |

### 1.3 主题切换

应用支持亮色/暗色主题切换：

- 点击左侧导航栏底部的 **主题切换按钮**（🌙 / ☀️）
- 首次启动时自动跟随系统主题设置
- 切换后会记住用户偏好，下次启动时保持所选主题 |

## 2. 导入文件夹

### 2.1 添加文件夹

1. 切换到 **浏览器** 页
2. 顶部工具栏点击 **`+ 添加文件夹`**
3. 选择包含照片的目录（支持子目录递归）
4. **扫描确认弹窗**：选择文件夹后会弹出确认窗口，显示扫描路径
5. 点击 **开始扫描**，应用使用多核并行计算自动扫描 + 提取 EXIF + 入库。

> 💡 **Windows 用户**：v2.2+ 扫描核心已迁移到 Rust FFI，MD5、EXIF、缩略图解码等计算在原生多线程中运行，导入速度更快、内存占用更低。其他平台仍保持 Dart 实现，功能完全一致。

扫描期间，内容区顶部会显示 **进度条面板**：

- **收起状态**：细进度条 + 当前文件名，不影响浏览
- **展开状态**（点击 ▲）：显示已新增/重复/速度/剩余时间，以及**暂停/停止**按钮
- 扫描进行时可以**继续浏览**已有照片，无需等待完成

> 重复添加同一文件夹不会重复入库；已删除的文件会自动标记为「缺失」。

### 2.2 拖拽导入

直接把文件夹 / 多张图片 **拖到浏览器窗口** 即可触发导入。

### 2.3 支持的文件格式

- **图片**：`jpg / jpeg / png / heic / heif / webp / tiff / bmp / gif`
- **视频**：`mp4 / mov / mkv / avi / wmv / webm / m4v / flv / mts / 3gp`

## 3. 图片浏览器

### 3.1 视图模式

左侧侧边栏可切换 **时间轴 / 标签 / 文件夹** 三种组织方式：

- **时间轴**：按拍摄年月分组，左侧显示月份列表或日历视图（可切换），点击可快速跳转到对应月份的媒体
- **标签**：按标签聚合，方便按主题查找，支持创建和管理标签
- **文件夹**：保持原目录结构，支持树形展开/折叠

### 3.2 显示密度

工具栏右侧可切换：

- **宫格图标**（4 列）：方便查看缩略图细节
- **列表视图**：紧凑显示，便于快速浏览

### 3.3 单张操作

- **单击**：打开大图预览
- **右键**：弹出操作菜单（查看 EXIF / 编辑标签 / 删除 / 在文件管理器中显示 / 复制路径）

### 3.4 大图预览

- **左右方向键** / **左右按钮**：切换上一张 / 下一张
- **`Esc`**：关闭预览
- **工具栏**：可对当前图打标签、删除、查看拍摄位置、打开所在文件夹

#### 3.4.1 缩略图清晰度智能切换

应用采用**双清晰度策略**自动平衡显示效果与内存占用：

- **缩略图模式**（默认）：300 像素宽，节省内存
- **原图模式**：1920 像素宽，停留 ≥ 20 秒后自动升级
- 升级后右下角显示"原图"小角标
- 离开视口后自动降级回缩略图

#### 3.4.2 按钮布局（v2.1+）

为解决"原图很矮时翻页按钮和全屏按钮重叠"的问题，按钮布局已重构：

- **左右翻页按钮**：固定在图片容器**外侧**（Row 边缘），永远垂直居中
- **全屏按钮**：固定在图片容器**底部居中**
- **顶部工具条**：关闭 / 文件名 / 打开所在文件夹
- **底部提示**：永远在屏幕底部，与翻页按钮互不干扰

### 3.5 视频播放

视频文件（`mp4 / mov / mkv / avi / wmv / webm / m4v / flv / ts / mts / 3gp`）支持内置播放器：

- **单击视频区域**：播放 / 暂停
- **底部进度条**：拖拽定位到任意时间点
- **播放时间**：显示当前播放位置 / 总时长
- **左右方向键**：切换上一个 / 下一个媒体文件

## 4. 多选与拖选

> 整理 / 删除 / 批量打标签时使用。

### 4.1 进入多选

- **长按**任一缩略图即可进入多选模式
- 进入多选后，**单击**可添加/取消选中

### 4.2 全选 / 反选

进入多选状态后，工具栏出现：

- **全选**：选中当前可见的全部缩略图
- **反选**：翻转每张图的选择状态

### 4.3 鼠标拖选（框选）

按住鼠标左键在网格上 **拖动**，可快速框选一片区域。
拖动过程中，已选中的会 **保持选中**，未选中的会 **被加入**；再次拖过同一张图则会 **取消选中**。

### 4.4 批量操作

选中后顶部出现操作栏：

- **打标签** – 给选中项一次性打上 / 取消标签
- **删除** – 软删除（不删除磁盘原文件）
- **移动 / 导出**（如有启用）

## 5. 标签管理

### 5.1 标签侧边栏

浏览器左侧的 **标签面板** 集成创建、删除、改色功能，无需单独的标签管理页。

### 5.2 创建标签

1. 点击面板底部 **`+ 新建标签`**
2. 输入标签名（可空 – 自动生成「未命名」）
3. 选择颜色：
   - **16 色标准色板**：快速选择预设颜色
   - **自定义调色盘**：HSV 选色器 + HEX 颜色码输入

### 5.3 编辑标签

在标签列表上 **右键** → 编辑：
- 修改标签名称
- 更改标签颜色

### 5.4 删除标签

在标签列表上 **右键** → 删除。已关联到此标签的照片会自动解绑。

### 5.5 给照片打标签

- 单张：右键 → 标签 → 勾选/取消标签
- 批量：多选后 → 顶部 **打标签** → 勾选/取消标签

### 5.6 按标签筛选

点击左侧标签面板中的标签，即可筛选出带有该标签的所有照片。

### 5.7 颜色含义

颜色仅用于视觉区分，**不影响搜索**。搜索按标签名匹配。

## 6. EXIF 与拍摄信息

### 6.1 自动提取

导入图片时自动读取：

- 拍摄时间、相机品牌 / 型号、镜头
- 光圈、快门、ISO、焦距
- **GPS 经纬度**
- 软件信息

### 6.2 GPS → 省市县（**离线**）

> 应用不会调用任何地图厂商 API，所有反查全部在本地完成。

- 提取经纬度后，使用内置的 `china_regions.json` 多边形数据
- 用「射线法 PNP + 省级中心点回退」判断属于哪个省
- 在同省内找最近的城市中心（Haversine 距离）
- 结果写入 `exif_data.province / city / cityName` 字段
- 仪表盘的「拍摄城市分布」图表就是基于这个字段

### 6.3 全库重新分析

如果你修改了 `assets/data/china_regions.json`，可以：

1. 浏览器顶部 → **更多 (⋮)** → **全库重新分析区域**
2. 应用会遍历所有带 GPS 的 EXIF 行，**重新反查并写回**

## 7. 仪表盘

四张统计卡 + 两个图表 + 标签词云。

### 7.1 统计卡

| 卡片 | 含义 |

| 媒体总数 | 未软删的照片数 |

| 总占用空间 | 所有未软删文件的合计大小 |

### 7.2 文件大小分布（饼图）

按尺寸分桶：

- < 1 MB
- 1–5 MB
- 5–20 MB
- 20–100 MB
- > 100 MB

帮你快速识别「空间杀手」是哪些照片 / 视频。

### 7.3 拍摄城市分布（柱状图）

按 EXIF 解析出的「省 / 市」聚合前 10 名。

**💡 新功能：照片地图视图**

- 地图上用圆点标记拍摄城市，**圆点大小 = 该城市照片数量**

- 点击城市圆点，底部会显示城市详情和 **「查看照片」** 按钮

### 7.4 标签词云

按使用频次显示。字号越大 = 关联照片越多。

> 软删除 / 导入新文件夹后，仪表盘自动刷新。

## 8. 搜索与筛选

> 浏览器顶部搜索栏支持 **6 种条件**，可任意组合。

### 8.1 条件列表

| 条件 | 输入 | 匹配规则 |
| --- | --- | --- |
| 📝 名称 | 关键词 | 文件名包含（不区分大小写） |
| 📷 设备 | 关键词 | 相机品牌 + 型号模糊匹配 |
| 📍 城市 | 关键词 | 城市 / 省份模糊匹配 |
| 📅 拍摄时间 | 起止日期 | 落在区间内 |
| 🏷 标签 | 多选 | 必须 **同时** 命中所有选中标签（AND） |
| 🎬 媒体类型 | 单选 | 仅图片 / 仅视频（默认不过滤） |

### 8.2 过滤 Chip

每个条件会显示为胶囊形状的 Chip，可单独移除（点击 `×`）。
点击搜索栏右侧的 **清空** 一键重置。

### 8.3 时间轴 / 标签模式下的搜索

- **时间轴模式**：搜索会过滤可见的月份
- **标签模式**：搜索会过滤可见的标签聚合
- **文件夹模式**：搜索条件照常生效，但 **不显示文件夹选择状态**

### 8.4 搜索结果无数据

- 检查「标签」是否过严（多标签是 AND 关系）
- 确认时间区间是否正确
- 城市搜索留空时 = 不过滤

## 10. 数据存储位置

### 10.1 应用数据（数据库）

数据库和缩略图缓存都保存在**应用安装目录**下的 `data/` 子文件夹中：

| 平台 | 真实路径 |
| --- | --- |
| **Windows** | `安装目录\data\pixelvault.db` |
| **Linux** | `安装目录/data/pixelvault.db` |
| **macOS** | `安装目录/data/pixelvault.db` |

> 💾 在 Windows 资源管理器里打开应用安装目录，进入 `data` 文件夹即可看到 `pixelvault.db` 主库文件，以及 `pixelvault.db-shm` / `pixelvault.db-wal` 两个 SQLite 临时文件（**不要手动删除**）。

> 📁 缩略图缓存保存在 `data/thumbnails/` 目录下，按文件 MD5 首字母分目录存储，便于快速查找和管理。

### 10.2 安装目录

打包后的安装文件夹（与 `pixelvault.exe` 同级）会包含：

- `pixelvault.exe` – 主程序
- `USER_MANUAL.md` – 本使用手册（首次启动时由应用自动生成）
- `sqlite3.dll` – 内嵌的 SQLite 引擎（无需用户安装）
- 各种 `*.dll` – Flutter 运行时依赖
- `data/` – 数据库和缩略图缓存目录

### 10.3 照片原文件

**应用永远不会移动 / 删除你的原始照片**。
软删除仅在数据库中标记 `is_deleted = true`，原始文件保持原样。

### 10.4 完全卸载

1. 退出应用（点击窗口关闭按钮，弹出确认对话框后自动保存数据库并退出）
2. 控制面板 → 程序 → 卸载 PicGuide
3. （可选）手动删除安装目录下的 `data/` 文件夹清空数据库和缓存

## 11. 常见问题

### Q1：导入后照片没出现？

检查：

1. 文件后缀是否在支持列表内
2. 文件夹权限是否可读
3. 是否被 **软删除** 过滤了（仪表盘「媒体总数」会减少）

### Q2：城市显示为「未知」？

- 部分照片本身没有 GPS（如截图、扫描件）
- 极地 / 境外：内置 JSON 仅覆盖中国境内
- 解决：浏览器 → 全库重新分析区域（不会修改你的数据，仅重算）

### Q3：标签颜色不对？

应用使用 `Color(0xFF + HEX)` 写入。**HEX 必须 6 位**，例如 `#5B8DEF`。
如果输入 `#ABC`，请补全为 `#AABBCC`。

### Q4：删除的照片找不回来？

- 软删除：30 天内可从回收站恢复（如果有）
- 硬删除：照片 **未真正删除**，原文件还在磁盘上
- 找回方法：在 `安装目录\data\pixelvault.db` 用任意 SQLite 客户端，把 `media_items.is_deleted` 改为 0

### Q5：能否远程 / 跨设备使用？

不支持。**PicGuide 是纯本地应用**：
- 没有云端账号
- 没有同步功能
- 所有数据都只在这台电脑

### Q6：数据库会不会无限增长？

不会。SQLite 单库 ≤ 几 GB 时性能良好。
软删除 / 软清理不会压缩数据库；如需彻底压缩：
```sql
VACUUM;
```

### Q7：原图很矮时翻页按钮和全屏按钮重叠？

**v2.1 已修复**：
- 翻页按钮改为 Row 外侧（垂直居中），不再叠在图片上
- 全屏按钮固定在图片底部居中
- 即使原图尺寸为 100×3000 这种极端比例，三个按钮也不会相互重叠

### Q8：缩略图卡顿 / 内存占用高？

**v2.1+ 已实施三层防御**：
1. **统一 PixelThumb 组件**：大图/中图/列表三种模式共用同一套缩略图逻辑
2. **视口热区 + 双清晰度**：冷区不触发 IO；热区先显示 300px 缩略图，停留 20s 升级 1920px 原图
3. **ImageCache LRU**：默认 300 项 / 200 MB 容量上限，超过自动驱逐

**手动调优**（开发者）：
- `lib/main.dart` → `imageCache.maximumSizeBytes`
- `lib/src/ui/browser/widget_pixel_thumb.dart` → `kFullCacheWidth`

### Q9：扫描时出现 `generateThumbnail failed: RangeError` 警告？

**原因**：纯 Dart 的 `image` 包对 9-patch、损坏的 WebP 等异常格式解码失败。
**影响**：被 try/catch 兜底，**不会**阻塞扫描。这些文件的缩略图会走原图加载兜底，仍能正常显示。

### Q10：Windows 扫描比旧版本快很多，是怎么做到的？

**v2.2+ 扫描核心使用 Rust FFI**：

- MD5、EXIF 提取、缩略图解码等 CPU 密集型操作放到 Rust 原生线程池中并行执行
- 缩略图解码采用 64 KB 流式分块，避免一次性读入大文件
- 相同任务下，Windows 桌面端的扫描速度显著提升，同时内存占用更低

> 其他平台（Linux / macOS / 移动端）仍保持 Dart 实现，功能完全一致，无需额外配置。

| 操作 | 方式 |
| --- | --- |
| 进入多选模式 | **长按**缩略图 |
| 添加/取消选中 | 多选模式下**单击** |
| 框选多张 | **鼠标拖动**画选区 |
| 右键菜单 | **右键点击**缩略图 |
| 关闭弹窗 | 点击右上角**关闭按钮** |

> 注：键盘快捷键（如 Esc、Delete、Ctrl+F 等）暂未实现，后续版本会逐步添加。
